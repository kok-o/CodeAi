# CodeAI

**Бағдарламалауды үйренуге арналған және кодты интеллектуалды бағалайтын веб-қосымша**  
*Веб-платформа для обучения программированию с ИИ-репетитором и интеллектуальной оценкой кода.*

Это дипломный проект, представляющий собой образовательную платформу нового поколения. Платформа предназначена для помощи пользователям в освоении различных языков программирования при поддержке персонального наставника на базе искусственного интеллекта.

---

## О проекте / Жоба туралы

**Цель:** Создание современной и эффективной среды для обучения программированию. Искусственный интеллект выполняет роль персонального репетитора: анализирует решения студентов, предлагает оптимальные подходы к написанию кода, выявляет синтаксические и логические ошибки, а также формирует индивидуальные траектории обучения.

**Мақсаты:** Жасанды интеллект жеке репетитор рөлін атқаратын бағдарламалауды үйренуге арналған заманауи және тиімді орта құру. ЖИ студенттердің шешімдерін талдайды, код жазудың оңтайлы тәсілдерін ұсынады, синтаксистік және логикалық қателерді анықтайды, сондай-ақ жеке оқу траекторияларын қалыптастырады.

---

## Основные возможности / Негізгі мүмкіндіктер

- **Интерактивная среда разработки (IDE):** Написание, редактирование и выполнение кода непосредственно в браузере на базе движка Monaco Editor.
- **Мультиязычность:** Поддержка выполнения кода на Python, JavaScript, C++, Java, Go, Rust и других современных языках программирования.
- **ИИ-Репетитор (AI Mentor):** Интеллектуальный помощник, который помогает диагностировать ошибки и объясняет сложные алгоритмические концепции. Репетитор спроектирован так, чтобы направлять студента к решению, а не давать готовые ответы, способствуя развитию навыков самостоятельного решения проблем.
- **Интеллектуальная оценка (Smart Evaluation):** Автоматизированная система проверки кода, использующая изолированную песочницу (Sandbox) для безопасного выполнения скриптов, прогона модульных тестов и статического анализа качества кода.
- **Адаптивное обучение:** Генерация персонализированных учебных планов нейросетью на основе метрик успеваемости, выявленных пробелов в знаниях и общего уровня подготовки пользователя.
- **Система мотивации и геймификации:** Интегрированная система достижений, уровней и рейтингов (от начального уровня до продвинутых лиг, таких как Diamond).
- **Ролевая модель:** Изолированные личные кабинеты для различных групп пользователей:
  - Студенты (Обучение и практика)
  - Преподаватели (Teacher Studio для создания курсов и мониторинга)
  - Администраторы (Admin Dashboard для управления платформой)

## Стек технологий / Технологиялар стегі

### Frontend (Клиентская часть)
- **Фреймворк:** React.js в связке с бандлером Vite для быстрой сборки.
- **Анимации и интерфейс:** Framer Motion для плавных переходов.
- **Визуализация данных:** Recharts для построения графиков прогресса и аналитики.
- **Редактор кода:** Monaco Editor (движок, используемый в VS Code).

### Backend (Серверная часть)
- **Среда выполнения:** Node.js.
- **Веб-фреймворк:** Express.js.
- **Интеграция искусственного интеллекта:** Stream API для асинхронного взаимодействия с локальными или облачными LLM-моделями (включая интеграцию с Ollama).
- **Выполнение кода:** Изолированная среда выполнения (Code Runner) для безопасного тестирования пользовательских скриптов.

## Инструкция по локальному развертыванию (Local Setup)

1. **Клонирование репозитория:**
   ```bash
   git clone https://github.com/kok-o/CodeAi.git
   cd CodeAi
   ```

2. **Установка зависимостей:**
   Необходимо установить npm-пакеты отдельно для клиентского приложения и серверного API.
   ```bash
   # Установка зависимостей клиента
   npm install

   # Установка зависимостей сервера
   cd server
   npm install
   cd ..
   ```

3. **Конфигурация переменных окружения:**
   Создайте конфигурационные файлы `.env` в корневой директории проекта и в директории `server`. Укажите в них необходимые параметры среды: порты, ключи доступа к API LLM и параметры подключения к базе данных. *Примечание: файлы .env исключены из системы контроля версий в целях безопасности.*

4. **Запуск приложения:**
   Для одновременного запуска клиентской и серверной частей в режиме разработки (при наличии настроенного concurrently скрипта):
   ```bash
   npm run dev
   ```
   
   Альтернативный вариант раздельного запуска:
   - Терминал 1 (Клиент): `npm run dev`
   - Терминал 2 (Сервер): `cd server && npm start`

5. **Доступ к приложению:**
   После успешного запуска откройте веб-браузер и перейдите по адресу `http://localhost:5173` для работы с платформой.
